搭建Mac本地环境

安装Hexo

参考这篇Hexo官方中文文档安装好Node.js、Git和Hexo。

注意：
所有命令请在Mac终端中运行，如果出现权限不够的情况，可使用sudo su -命令切换到root超级用户（但是不建议使用root用户）。

开始安装

在终端执行一下命令，在你的Mac上新建一个网站专用文件夹：

$ hexo init <更改此处为你的网站文件夹>
$ cd <更改此处为你的网站文件夹>
$ npm install

注意：
在安装过程中可能会出现「卡住」的状态，别担心，很有可能是网络问题，使用Ctrl+C终止当前运行的程序，参考这个链接更改npm为国内的软件源。

然后键入一下命令测试Hexo是否正常：

hexo s --debug //默认本地地址localhost:4000，可以看到调试信息

同步Hexo到Github

上面你看到的只是在本地机器上的测试，还需要下面的步骤来将网站内容上传到Github：

npm install hexo-deployer-git --save

安装完成后打开本地博客目录_config.yml文件，更改文本最下方deploy节点为以下状态；

deploy:
  type: git
  repo: https://github.com/youlookwhat/youlookwhat.github.io.git
  branch: master

保存配置后，确认本地可以正常工作后，键入一下命令，上传到github：

hexo clean %% hexo g %% hexo d

这时你就可以在username.github.io看到你的博客了，如果在你添加文章或者做了任何相关的修改后，保证网站正常工作后，使用上面的一行代码上传到Github。

Next主题安装与配置

安装、启用Next

可以参考Next作者编写的参考文档来安装。

调试代码后可以使用以下一行命令打开本地测试：

hexo clean %% hexo g && hexo s --debug

测试无误后使用以下代码上传到Github：

hexo clean %% hexo g %% hexo d

注意：
这两行代码可以当作日常维护时使用，测试代码在运行时会实时编译你的更改，方便查看效果，完成所有更改之后上传代码。

注意：
日常新建、编辑自己文章建议使用MWeb来编辑，详细的设置方法可以参考这篇文章

常见问题

  • 本地调试时如果遇到浏览器缓存没有及时清理的问题，可以使用Alt+Commamd+R快捷键来忽略缓存刷新页面。
